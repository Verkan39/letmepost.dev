The letmepost.dev MCP server lets AI agents publish to social platforms in one tool call. Works with Claude Code, Cursor, Cline, Claude Desktop, and anything else that speaks the [Model Context Protocol](https://modelcontextprotocol.io).

The MCP server is a thin wrapper over the REST API. Same auth, same idempotency, same error envelope. The agent never sees an OAuth token.

## Install

The fastest path is `npx`. Set your API key and the server boots over stdio.

<CodeGroup>

```bash Claude Code
claude mcp add letmepost npx @letmepost/mcp --env LMP_API_KEY=lmp_live_...
```

```json Cursor / Cline / Claude Desktop
{
  "mcpServers": {
    "letmepost": {
      "command": "npx",
      "args": ["@letmepost/mcp"],
      "env": { "LMP_API_KEY": "lmp_live_..." }
    }
  }
}
```

```bash Bare npx
LMP_API_KEY=lmp_live_... npx @letmepost/mcp
```

</CodeGroup>

Generate an API key at [dashboard.letmepost.dev](https://dashboard.letmepost.dev). The key inherits the profile scope it was minted with so the agent can't see accounts outside that profile.

## Configuration

| Variable        | Required | Default                     | Description                                       |
| --------------- | -------- | --------------------------- | ------------------------------------------------- |
| `LMP_API_KEY`   | yes      |                             | Bearer token from the dashboard                   |
| `LMP_API_BASE`  | no       | `https://api.letmepost.dev` | Override when pointing at a self-hosted instance  |

## Tools

### `publish_post`

Publishes to one or more connected accounts in a single call. Fans out in parallel, returns a batch envelope with per-target results.

**Input**

```json
{
  "text": "Shipping the MCP server today.",
  "targets": [
    { "platform": "twitter" },
    { "platform": "bluesky" },
    { "platform": "threads" }
  ]
}
```

Each target is either `{ accountId }` (explicit) or `{ platform }` (auto-resolves the org's single connected account for that platform under the API key's profile). Per-target `text` overrides the top-level default.

**Output**

```json
{
  "id": "pst_01HX...",
  "status": "published",
  "createdAt": "2026-05-18T12:34:56Z",
  "results": [
    { "accountId": "acc_...", "platform": "twitter", "postId": "...", "status": "published", "uri": "https://x.com/.../status/..." },
    { "accountId": "acc_...", "platform": "bluesky", "postId": "...", "status": "published", "uri": "at://did:plc:.../app.bsky.feed.post/..." },
    { "accountId": "acc_...", "platform": "threads", "postId": "...", "status": "published", "uri": "https://www.threads.net/..." }
  ]
}
```

Batch `status` is `published`, `partial_failed`, `failed`, or `queued`. Failures surface inside `results[i].error` with `code`, `rule`, `platform`, `platformResponse`, `remediation`, `docUrl`, and `ruleUrl` so the agent can decide what to do next.

The MCP server stamps a unique `Idempotency-Key` on every call automatically. If the agent retries after a transport blip, the API replays the same batch instead of double-posting.

### `list_accounts`

Lists the connected accounts the API key can see. Useful when the agent needs an explicit `accountId` or wants to confirm what's connected before composing a multi-target call.

**Input**

```json
{ "platform": "bluesky" }
```

`platform` is optional. Omit to list every connected account on the profile.

**Output**

```json
[
  { "id": "acc_...", "platform": "bluesky", "displayName": "you.bsky.social", "createdAt": "..." }
]
```

## Example: cross-post from Claude Code

Once installed, the agent can compose a multi-target post in plain English:

> "Post the text 'Shipping the MCP server today' to X, Bluesky, and Threads."

Claude calls `publish_post` with `{ targets: [{platform: "twitter"}, {platform: "bluesky"}, {platform: "threads"}], text: "Shipping the MCP server today" }` and renders the batch envelope back. One call, three platforms, identical error contract across all of them.

## Self-host

Self-host bypasses every platform approval gate. Point the MCP server at your own deployment:

```sh
LMP_API_KEY=lmp_live_... LMP_API_BASE=https://api.yourdomain.com npx @letmepost/mcp
```

See [self-host quick-start](/self-host/quick-start) for the full deployment walkthrough.

## What's not in the MCP server yet

- `get_post` and `list_posts` (read-side endpoints) are on the roadmap. The REST API exposes them today.
- Hosted MCP at `mcp.letmepost.dev/mcp` (HTTP transport, OAuth instead of API key) is planned. For now the local `npx` install with stdio is the supported path.

File issues or feature requests at [github.com/letmepost/letmepost.dev](https://github.com/letmepost/letmepost.dev).
